**Регламент обновления:** После каждого Спринта или по запросу Координатора

## 1. Текущее состояние проекта (апрель 2026)

### 1.1. Выполненные спринты

| Спринт | Тема | Статус |
|--------|------|--------|
| №1 | Фундамент (абстрактные биржи, multi-coin, метрики) | ✅ Завершён |
| №2 | Автооптимизация параметров (Optuna, триггеры) | ✅ Завершён |
| №3 | Market Regime + Risk Manager + 3 стратегии | ✅ Завершён |
| №4 | Стабилизация, reload_flag, UX | ✅ Завершён |
| №5 | Новые биржи + Общая библиотека | ✅ Завершён |
| №6 | Инфраструктура тестирования | ✅ Завершён |

### 1.2. Текущая функциональность

- Торговля на Bybit (CRYPTO_BOT), Тинькофф (TINKOFF_BOT), Мосбирже (MOEX_BOT)
- Multi-coin (один бот → много символов)
- Автооптимизация параметров (Optuna)
- Градуированные триггеры (0/1/2) с защитой от ложных срабатываний
- Market Regime Filter (ADX + ATR, приоритет HIGH_VOLATILITY)
- Risk Manager + Volatility Guard (восстановление после Halt)
- 3 стратегии (MA Crossover, Bollinger, SuperTrend)
- Out-of-sample проверка (overfit_ratio)
- Telegram управление с префиксами (crypto_, tinkoff_, moex_)
- reload_flag (применение параметров без перезапуска)
- Инфраструктура тестирования (L1/L2/L3, pytest, ручной чек-лист)

### 1.3. Текущие ограничения

- Старые single-coin боты ещё работают (технический долг)
- Нет автоматических еженедельных отчётов (ручной режим)
- Нет накопленной статистики реальной торговли
- Prediction Module не реализован (отложен до V2.0)

## 2. Ближайшие этапы (V1 — Стабилизация и реальная торговля)

### Этап 1: Демо-торговля (Спринт №7)

**Тип:** Эксплуатационный этап (не разработка)

| Задача | Длительность | Ответственный |
|--------|--------------|---------------|
| Запуск демо-торговли на всех трёх ботах | 1 день | Координатор (по инструкции Архитектора) |
| Ежедневный мониторинг логов и метрик | 7-14 дней | Координатор + Архитектор (анализ) |
| Настройка порогов триггеров под реальные данные | По мере необходимости | Архитектор (после консультации с Консультантом) |
| Сбор статистики, выявление проблем | 14 дней | Архитектор |
| Еженедельные отчёты (ручной режим) | Каждый понедельник | Архитектор (инструкция) → Координатор (отправка) |

**(*) Уточнение по демо-торговле:**

- **CRYPTO_BOT:** используется официальный Testnet Bybit (имитация реальной торговли без риска)
- **TINKOFF_BOT:** у Тинькофф нет официального демо-счета. Под «демо-торговлей» понимается **торговля на реальном счете минимальными лотами** (например, 1 акция) с целью тестирования логики, логирования и риск-менеджмента. Рекомендуемый капитал — 5000-10000 рублей.
- **MOEX_BOT:** аналогично Тинькофф, используется **реальный счет с минимальными лотами** (1 фьючерсный контракт). Рекомендуемый капитал — 10000-20000 рублей.

**Критерий перехода к следующему этапу:** Система стабильно работает на демо-счетах 2 недели без критических ошибок.

**Ориентировочная длительность:** 2-3 недели

---

### Этап 2: Реальная торговля минимальным капиталом (Спринт №8)

**Тип:** Эксплуатационный этап

| Задача | Длительность | Ответственный |
|--------|--------------|---------------|
| Переключение ботов на реальные счета (минимальный капитал) | 1 день | Координатор |
| Ежедневный мониторинг реальных сделок | 14 дней | Координатор + Архитектор |
| Анализ проскальзывания, комиссий, задержек | 7 дней | Архитектор + Консультант |
| Корректировка риск-параметров | По необходимости | Архитектор |
| Еженедельные отчёты по реальной торговле | Каждый понедельник | Архитектор → Координатор |

**(*) Примечание:** На этом этапе для Тинькофф и Мосбиржи используется тот же реальный счет, что и на этапе демо-торговли, но с увеличенным капиталом до комфортного минимума (например, 50000-100000 рублей для акций и фьючерсов).

**Критерий перехода к следующему этапу:** Система показывает стабильную (пусть и небольшую) положительную динамику или околонулевую с контролируемыми просадками.

**Ориентировочная длительность:** 3-4 недели

---

### Этап 3: Эксплуатация и накопление данных (после Спринта №8)

**Тип:** Эксплуатационный этап (без активной разработки)

| Задача | Длительность | Ответственный |
|--------|--------------|---------------|
| Торговля на реальных счетах без новых фич | 3-4 недели | Координатор (запуск), Архитектор (мониторинг) |
| Сбор метрик для анализа эффективности | Весь период | Архитектор |
| Выявление слабых мест в эксплуатации | Весь период | Координатор + Архитектор |
| Еженедельные отчёты с рекомендациями | Каждый понедельник | Архитектор → Координатор |

**Критерий перехода к V2:** Накоплена достаточная статистика (минимум 100 сделок на каждого бота) для принятия решения о дальнейшем развитии.

**Ориентировочная длительность:** 3-4 недели

### Этап 4: Миграция single-coin ботов (Спринт №9)

| Задача | Длительность | Ответственный |
|--------|--------------|---------------|
| Перенос символов старых ботов в CRYPTO_BOT | 3 дня | Разработчик |
| Тестирование (L2/L3) | 2 дня | Тестировщик |

| Удаление кода старых ботов (архивация) | 1 день | Координатор |

**Критерий завершения:** Все single-coin боты отключены, их символы торгуются через CRYPTO_BOT.

**Ориентировочная длительность:** 1.5-2 недели
